Welcome



Thanks for visiting my page! My name is Torsten Bergmann - but some call me astares as this is my internet nickname for many years.

I'm a father, professional IT architect, coder, software engineer, advisor, hitchhiker, and geek — all rolled into one.

I'm an engineer working with and in IT and I hold a degree in Computer Science from the Technical University of Ilmenau, specializing in software engineering and telematics. As you will find here I am a strong advocate of the OOP programming paradigm, dynamic languages, metadata-driven software technology, and the model-driven approach. My thesis from 1999 focused on component-oriented software development (CORBA, COM/COM+, Java EJB) so I love modular software and systems.

I have around 30 years of experience, worked as

  • Software developer
  • Consultant
  • Project manager
  • Software and cloud architect
  • IT manager and global enterprise architect

with interesting challenges and wonderful colleagues in international setups. My solida background and extensive knowledge in development, operations but also (cyber)security has enabled me to manage critical infrastructure across various industries, including finance, insurance, logistics, and transport.

And I absolutely love it - for me IT is a passion.

Some projects

This is a selection of some of my open source projects. You can find more below as well as on GitHub.

astares/Pharo-OS-Windows My windows system library for Pharo wrapping most of the Windows native APIs so you can easily use Windows features in a proper object-oriented way. Comes with extensive set of tests. Handcraftet code using the Unified Foreign Function Interface (UFFI)
astares/Pharo-Docker Containerization is on the rise. So I wrote a UI tool and library to easily work with docker containerization from within Pharo. If you work with docker images you might find it useful. Also allows to search the docker registry using Spotter.
astares/Pharo-XMI This utility allows you to read, display and process files in the XML Metadata Interchange (XMI) standard as defined by Object Management Group (OMG) for exchanging metadata information. It is especially useful for modeling and code generation.
astares/Pharo-Pomodoro This small utility represents my implementation of a Pomodoro timer following the Pomodoro Technique (a time management method). Check my video for more.
astares-nucleus/staruml-pharo Extensions for WhiteStar UML tool to be able to more easily model real Pharo domain models.
astares/Pharo-Teachable A teachable object for Pharo - that you can teach dynamically how to respond. Very useful for mocking in tests and a nice example what is possible in dynamic programming languages.
astares/Pharo-I18N I18N is short for Internationalization. This is a a useful project and tool to help you master translations for internationalized applications.
astares/Tealight Tealight is my minimalistic web and REST framework for Pharo. More lively and interactive than Spring, Quarkus, Micronaut or Django could ever be.
astares/Pharo-Temple A minimalistic template engine that I wrote to be used in web development
astares/Pharo-Chrome A library I wrote for remotely driving the Chrome webbrowser - useful for web based testing
astares/Pharo-VirtualBox Some wrapper to work more easily with VirtualBox virtualization
astares/Pharo-INIFile Portable Parser for .INI files so they are not only usable on Windows but also on Linux and Mac. Useful for config management and settings.
astares/Pharo-PowerShell A bridge for PowerShell scripting from within Pharo.
Pharo-WebSockets An example on how interactively and lively one can work with WebSockets with Zinc HTTP framework and using Pharo.
Pharo-PartialDate PartialDate implementation for Pharo to deal in an application with (yet) incomplete entered dates
Pharo-Currencies Better support for currencies to use and interact with them as real objects.

    Pharo on a RaspberryPi machine

    "Write once run anywhere" (WORA) - the ability to run on different devices was already present in Smalltalk before the programming language Java popularized it. Over the years more and more computing power became available. As an IT guy I find it interesting to experiment with software on various platforms and devices (TV, smartphone, smartwatch). In this video you see me running Pharo on a custom done Raspberry Pi. I own several Pi machines and also use them to host headless server apps like "Swiki"

    If you run Pharo on a Pi you might also want to check out my special support package for Raspbian: https://github.com/astares/Pharo-OS-Raspbian

    Pharo with virtual screens

    As a developer you often have many windows open - typically you organize them on virtual desktop screens. While this is pretty standard on Linux desktop managers Pharo unfortunately did not have support for it. So I provided an implementation as a tool to the community based on World switching and Pharo 5 version. It also integrates nicely into spotter.

    Astares UML Support for Pharo

    instead of hardcoding it is often useful to keep (design) info inside of a model created with the help of the Unified Modelling Language (UML). The included meta information can be used to generate code for a target platform (following the classical approach) or keeping the meta-info available and changeable within the runtime.

    Especially the latter approach is some fascinating way to automize application building to a high degree - even beyond what is today known as low-code platforms. I have many ideas in my head for that topic and for a few I already wrote some nice tools like an XMI importer and several model runtime platforms. The goal is to even bootstrap the metamodel from a model description. For me this is interesting private research as it also shifts away the focus from code reuse towards reusing model parts and components.

    Remote controlling Chrome from Pharo

    When doing web development you might want to query informations from webpages or webservices, automize your processes based on web content or check if the contents a webpage or webapp is showing is proper and correct. So I wrote a framework for Pharo that allows you to remotely query and control the Chrome browser easily.

    Code and docu is available on GitHub at https://github.com/astares/Pharo-Chrome

    Serve static web content from Pharo using Teapot

    In web based projects you might have the need for development reasons to quickly serve static content or pages. I guess you know tools like "Liveserver" from VisualStudio Code and other. The video demonstrates that similar results could easily achieved with Pharo and the Teapot web framework.

    Code and docu is available on GitHub at https://github.com/zeroflag/Teapot

    Serve dynamic web content from Pharo using Tealight

    When I contributed to the amazing Teapot web framework I felt like something was missing. I wanted to increase the interactive nature and also have better support for the REST based approach. So I wrote Tealight which is a web framework on its own and allows for instance rapidly prototype but also dynamically implement an own service or webpage.

    The Code for Tealight is available on my GitHub account at https://github.com/astares/Tealight together with an extensive documentation.

    Pharo OS-xxx series support

    While Pharo is platform independent on one side it is often necessary to access native features of the underlying platform. While initially different approaches for callouts/callbacks had been used (like FFI/Alien, then NativeBoost with assembler generation based on AsmJit) the Foreign Function interface has later been unified into UFFI and even allows threaded FFI (TFFI). For better support on platforms like Windows, iOS and different Linux distros I wrote specific packages:

    • https://github.com/astares/Pharo-OS-Windows
    • https://github.com/astares/Pharo-OS-OSX
    • https://github.com/astares/Pharo-OS-Unix
      • https://github.com/astares/Pharo-OS-Linux-Ubuntu
      • https://github.com/astares/Pharo-OS-Raspbian
      • http://smalltalkhub.com/OS/OS-Linux-CentOS

    Pharo OS-Linux-Ubuntu support

    While I work a lot on Windows I like and use Linux distros a lot for coding but also automation. Ubuntu is one of the few distros that provide many packages to their users. So why not spice it up with a more object oriented approach to allow for better and easy usage right from inside the Pharo lively environment.

    Code and docu is available on GitHub at https://github.com/astares/Pharo-OS-Linux-Ubuntu
